# 07-02 Hyperliquid Application Patterns and Lyquor

Related blog post: `blog/2026-07-02-hypercall-business-analysis/index.md`

## English

Hyperliquid is becoming more than a perp exchange. HyperCore is specialized trading infrastructure, HyperEVM adds a programmable application layer around it, and HyperCall shows how trading products can be built close to liquidity, account state, margin, and settlement. If this kind of business were implemented on Lyquor, the key difference is that developers could build the trading infrastructure itself: matching, clearing, risk, margin, liquidation, and oracle-driven settlement could become sequenced Lyquid network applications sharing state and coordinating through Lyquor runtime capabilities.

## Chinese

Hyperliquid 正在变成不只是 perp 交易所。HyperCore 是专用交易基建，HyperEVM 在它周围增加可编程应用层，HyperCall 则展示了交易产品如何贴近流动性、账户状态、保证金和结算来构建。假设用 Lyquor 来实现这类业务，关键差异是开发者可以构建交易基建本身：撮合、清算、风控、保证金、强平和 oracle-driven settlement 可以成为被统一排序驱动的 Lyquid network applications，共享状态，并通过 Lyquor runtime 能力协同运行。
